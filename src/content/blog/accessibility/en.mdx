# Web Accessibility (A11y) - Complete Guide

Web accessibility is the practice of creating websites that can be used by all people, regardless of their abilities. This is not only ethically right but also legally required in many countries.

## What is A11y?

A11y (Accessibility) includes:

- Visual limitations
- Hearing limitations
- Motor limitations
- Cognitive limitations

## Accessibility Standards

### WCAG 2.1

Guidelines include:

- **Perceivable**: Content must be perceivable
- **Operable**: Interface must be operable
- **Understandable**: Information must be understandable
- **Robust**: Content must be robust

### Compliance Levels

- **A**: Minimum level
- **AA**: Recommended level
- **AAA**: Highest level

## Practical Recommendations

### Semantic HTML

Use proper tags:

- `<header>`, `<nav>`, `<main>`, `<article>`
- Proper heading hierarchy
- ARIA attributes where necessary

### Keyboard Navigation

Ensure:

- Tab navigation
- Visible focus
- Logical element order
- Skip links

### Alternative Text

All images should have:

- Descriptive alt text
- Empty alt for decorative images
- Text alternatives for charts
