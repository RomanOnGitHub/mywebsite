# Структурный аудит проекта (Astro)
Дата: 2026-01-24

## Сводка
Проект в целом соответствует требованиям Astro 5.18+ и внутренним правилам. Критичных ошибок не обнаружено. Есть несколько мелких улучшений по структуре и документации.

## Проверено
- Конфиг Astro: корректно, без запрещенных паттернов.
- Content Collections: используется `loader: glob()`, без `reference()`.
- View Transitions: используется `<ClientRouter />`, без `<ViewTransitions />`.
- Breadcrumbs: присутствуют на индексных страницах разделов.
- Lifecycle cleanup: обработчики и синглтоны корректно очищаются.
- Запрещенные зависимости: `vis-network` не используется в коде.

## Замечания и улучшения
1. `src/integrations/graph-integration.ts`
   - Есть относительные импорты `../..` (допустимо для интеграций, но лучше унифицировать).
   - Идея: проверить, можно ли включить `@/` alias для node-контекста интеграций.

2. `src/components/`
   - Плоская структура папки компонентов.
   - Рекомендация (по правилам проекта): разнести на `ui/`, `sections/`, `layout/`, когда количество компонентов вырастет.

3. `src/pages/[lang]/graph.astro`
   - Файл крупный (≈1600+ строк), но уже использует отдельные модули в `src/utils/`.
   - При дальнейшем росте — выносить ответственность по модулям (filters/highlight/export/zoom/state).
